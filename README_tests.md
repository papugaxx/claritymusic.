# SAFE TESTS GUIDE

<!-- Коментар пояснює призначення наступного XML блока -->
## Как запускать тесты фронтенда

1. Открой папку frontend.
2. Выполни `npm install`, чтобы обновить зависимости и package-lock.
3. Выполни `npm run test`.

<!-- Коментар пояснює призначення наступного XML блока -->
## Как запускать тесты бэкенда

1. Открой папку backend.
2. Выполни `dotnet restore`.
3. Выполни `dotnet test CLARITY.music.Api.Tests/CLARITY.music.Api.Tests.csproj`.

<!-- Коментар пояснює призначення наступного XML блока -->
## Что покрывают тесты

Фронтенд:

* storage для auth процес
* storage для черновика регистрации
* валидацию загрузки
* маппинг сообщений auth feedback
* route guards с замоканным auth state

Бэкенд:

* нормализацию lookup-имен
* правила валидации треков
* валидацию адмінка трека
* правила безопасности media URL
* построение account-ссылок
* поведение поточний user service с замоканными зависимостями
