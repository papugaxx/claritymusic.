# CLARITY.music frontend

<!-- Коментар пояснює призначення наступного XML блока -->
## Назначение
Клиентская часть проекта CLARITY.music реализована на React + Vite и отвечает за пользовательский интерфейс музыкального веб-приложения.

Frontend обеспечивает:
- публичные и защищённые маршруты;
- отображение каталога, плейлистов, профиля и настроек;
- работу player bar и общих layout-компонентов;
- взаимодействие с backend API;
- обработку пользовательских сценариев для ролей User, Artist и Admin.

## Структура проекта
Ключевые каталоги:
- `src/app` — маршрутизация, layout и route guards;
- `src/pages` — страницы приложения;
- `src/components` — переиспользуемые UI-компоненты;
- `src/features` — прикладные модули и вспомогательная логика;
- `src/services` — работа с API;
- `src/context` и `src/contexts` — контексты состояния;
- `src/styles` — стили приложения.

Точка входа маршрутизации: `src/app/router.jsx`.

## Настройка API
Frontend получает адрес backend API в следующем порядке приоритета:
1. переменная окружения `VITE_API_BASE`;
2. `window.__CLARITY_API_BASE__`;
3. значение из `localStorage` по ключу `clarity:apiBase`;
4. same-origin для не локального окружения;
5. `http://localhost:5287` для локальной разработки.

Если backend запускается локально со стандартным профилем, отдельная настройка обычно не требуется.


VITE_API_BASE=http://localhost:5287

## Особенности работы frontend
- приложение использует cookie-based авторизацию;
- для изменяющих запросов используется CSRF-токен;
- данные о текущем пользователе и ролях подгружаются из backend;
- player bar является общей частью shell-интерфейса;
- часть маршрутов защищена через route guards.

## Что проверить после запуска
После старта frontend рекомендуется проверить:
- открывается ли landing page;
- работает ли регистрация и вход;
- подгружается ли главная страница `/app`;
- отображаются ли треки, плейлисты, профиль и настройки;
- корректно ли работает player bar;
- открываются ли artist и admin-разделы для соответствующих ролей.
